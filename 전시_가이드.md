## 1. 사전 요구사항

| 항목 | 내용 |
|---|---|
| OS | Windows 10 / 11 |
| Node.js | v18 이상 ([nodejs.org](https://nodejs.org) 에서 LTS 설치) |
| 브라우저 | Google Chrome (최신 버전) |
| 마이크 | USB 마이크 또는 내장 마이크 |
| 인터넷 | **최초 설치 시에만 필요** (이후 오프라인 운영 가능) |

> Node.js 설치 확인: 명령 프롬프트에서 `node -v` 실행 시 버전 번호가 나오면 OK

## 2. 최초 설치 순서

### Step 1 — 관리자 설정 스크립트 실행 (딱 1번만)

```
전시_설정_최초1회_관리자실행.ps1  →  우클릭  →  "PowerShell로 실행" (관리자 권한)
```

이 스크립트가 자동으로 처리하는 항목:

| # | 항목 | 효과 |
|---|---|---|
| 1 | Chrome 자동 업데이트 차단 | 전시 중 Chrome이 업데이트·재시작되지 않음 |
| 2 | 마이크 권한 자동 허용 | localhost:8080에서 마이크 팝업 없이 자동 허용 |
| 3 | Windows Update 35일 유예 | 전시 중 Windows가 자동 재시작되지 않음 |
| 4 | 절전 모드 / 화면보호기 비활성화 | LED 화면이 꺼지지 않음 |
| 5 | 부팅 자동 시작 등록 | 로그인 시 전시가 자동으로 시작됨 |

> **PowerShell 실행 오류 시**: Windows 검색에서 "PowerShell" 검색 → 우클릭 → "관리자로 실행" 후 아래 명령 입력
> ```
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope LocalMachine
> ```
> 이후 스크립트 재실행

---

### Step 2 — 최초 빌드 (첫 실행 시 자동으로 처리됨)

`전시_시작.bat`를 더블클릭하면:

1. `node_modules` 없으면 → `npm install` 자동 실행
2. `dist` 빌드 파일 없으면 → `npm run build` 자동 실행
3. 로컬 서버 시작 (http://localhost:8080)
4. Chrome 키오스크 모드로 자동 실행

**최초 실행은 빌드에 2~5분 정도 소요됩니다.**

---

### Step 3 — 마이크 권한 허용 (최초 1번)

정책 설정이 적용되더라도 Chrome 버전에 따라 **첫 실행 시 마이크 허용 팝업**이 뜰 수 있습니다.

- 팝업이 나타나면 **"허용"** 클릭
- 이후 재실행부터는 팝업 없이 자동 허용

---

### Step 4 — 재부팅 확인

재부팅 후 로그인하면 전시가 자동 시작되는지 확인합니다.

## 3. 전시 운영 방법

### 정상 운영 상태

- 앱이 시작되면 **자동으로 키오스크 모드**(전시 모드)로 진입합니다.
- 마이크가 자동으로 켜지며 소리에 반응하기 시작합니다.
- UI 버튼, 설정 패널 등은 모두 숨겨진 상태입니다.
- 무음 상태에서도 미세한 파티클이 생성되어 LED가 완전히 정지하지 않습니다.

### 자동 관리

| 기능 | 내용 |
|---|---|
| 24시간 자동 재시작 | 매 24시간마다 브라우저가 자동 새로고침 (메모리 누수 방지) |
| AudioContext 감시 | 5초마다 오디오 상태 확인, 정지 시 자동 복구 |
| 음성 인식 재연결 | Speech API 끊김 시 자동 재시작 (최대 지수 백오프) |

## 5. 전시 중 감도 조절

`Ctrl+Shift+K` → 세팅 모드 진입 → 하단 ⚙ 버튼

| 설정 | 범위 | 권장값 | 설명 |
|---|---|---|---|
| 마이크 감도 | 0.1 – 5.0 | **0.4** | 높을수록 작은 소리에도 반응 |
| 반응 임계값 | 0.005 – 0.3 | **0.06** | 낮을수록 배경 소음에도 반응 |

**전시 공간이 시끄러운 경우**: 감도 낮추고 임계값 높이기 (예: 감도 0.2 / 임계값 0.15)
**조용한 공간**: 기본값 유지 또는 감도 약간 높이기

## 7. 트러블슈팅

### 마이크가 반응하지 않음

1. `Ctrl+Shift+K`로 세팅 모드 진입
2. `Space` 또는 "마이크 시작" 버튼 클릭
3. Chrome 주소창 왼쪽 🔒 아이콘 클릭 → 마이크 → "허용" 확인

### 한국어 음성 트리거(사랑해, 안녕 등)가 동작하지 않음

Web Speech API는 인터넷 연결이 필요합니다. 인터넷 연결을 확인하세요.
오프라인 환경에서는 음성 트리거는 비활성화되지만, 기본 소리 반응 시각화는 정상 동작합니다.

### 화면이 까맣게 보임 (아무것도 안 나옴)

1. `Space`를 눌러 마이크 활성화 여부 확인
2. Chrome DevTools(F12) → Console 탭에서 오류 확인
3. `dist` 폴더 삭제 후 재빌드

### 서버가 이미 실행 중이라는 오류

이전 서버 프로세스가 남아 있는 경우입니다.

```
작업 관리자(Ctrl+Shift+Esc) → 세부 정보 탭 → node.exe 찾아서 종료
→ 전시_시작.bat 재실행
```

### PowerShell 스크립트 실행 오류

```powershell
# 관리자 PowerShell에서 실행
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope LocalMachine
```

### 자동 시작이 안 됨 (재부팅 후)

작업 스케줄러 확인:
```
작업 스케줄러 → 작업 스케줄러 라이브러리 → "AbstractEchoes_전시시작" 확인
없으면 전시_설정_최초1회_관리자실행.ps1 재실행
```

## 8. 수정 이력 (LED 전시 대응)

이 항목은 일반 개발 버전 대비 LED 전시용으로 변경된 내용입니다.

| 파일 | 변경 내용 | 이유 |
|---|---|---|
| `SoundCanvas.tsx` | 캔버스 해상도 1900×1200 → **1408×960** | LED 월 실제 해상도 일치 |
| `SoundCanvas.tsx` | 키오스크 모드 기본값 `false` → **`true`** | 시작부터 전시 모드 진입 |
| `SoundCanvas.tsx` | AudioContext watchdog 추가 (5초 주기) | 장기 가동 시 오디오 정지 자동 복구 |
| `SoundCanvas.tsx` | 24시간 자동 새로고침 타이머 추가 | 메모리 누수 방지 |
| `generativeEngine.ts` | 무음 시 ambient 파티클 추가 (30프레임마다 1개) | LED 픽셀 번인 방지 |
| `audioAnalyzer.ts` | `resumeIfSuspended()` 메서드 추가 | watchdog에서 AudioContext 복구 |
| `vite.config.ts` | preview 포트 8080 고정 | 개발/전시 포트 통일 |
| `전시_시작.bat` | 신규 생성 | 원클릭 전시 시작 자동화 |
| `전시_설정_최초1회_관리자실행.ps1` | 신규 생성 | 전시 환경 OS/브라우저 설정 자동화 |
